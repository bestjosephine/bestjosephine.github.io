1. 登录

首次使用需要输入 GitHub Personal Access Token（需要 repo 权限）。Token 仅保存在当前浏览器会话中，关闭页面后自动清除。

如果还没有 Token，点击登录页面的"点此创建"链接，会跳转到 GitHub 创建页面，勾选 repo 权限后生成即可。

2. 新建文章

  1. 填写标题，系统会自动生成 URL 路径（Slug）
  2. 可手动修改 Slug，系统会自动检测是否与已有文章冲突
  3. 选择日期（默认为今天）和填写标签
  4. 在编辑器中使用 Markdown 语法写正文
  5. 点击"发布"按钮，文章会通过 GitHub API 提交到仓库
发布后 GitHub Actions 会自动构建，约 1-2 分钟后文章即可在线访问。

3. 编辑文章

在"已发布文章"列表中点击"编辑"，文章内容会加载到编辑器中。编辑模式下 URL 路径（Slug）不可修改，以保证已有的链接不会失效。修改完成后点击"发布"即可更新。

4. Markdown 语法

编辑器支持标准 Markdown，工具栏提供了常用的格式化快捷按钮：

  • 加粗：**文本**
  • 斜体：*文本*
  • 标题：# 一级标题 ## 二级标题
  • 链接：[显示文字](URL)
  • 图片：![描述](图片URL)
  • 代码块：使用三个反引号包裹

点击工具栏的 Preview 按钮可以实时预览渲染效果。

5. 注意事项

  • Slug 只允许英文字母、数字和短横线，建议使用有意义的英文短语
  • 标签（Tag）用于文章分类，建议每篇文章设置一个标签
  • 图片需要先上传到图床或 GitHub，然后使用图片链接插入
  • Token 安全提示：不要在公共设备上保存 Token，使用完毕建议点击"登出"
